## 1. 文档目的

本文档用于描述 Court Kicks 篮球鞋租赁平台的产品定位、目标用户、核心功能、业务流程及非功能性需求，作为产品迭代、开发、测试和运营的基础依据。

---

## 2. 产品概述

### 2.1 产品背景

篮球鞋价格较高，且不同位置、打法对鞋款需求差异大。普通篮球爱好者希望以较低成本体验不同实战鞋款，避免购买后不合适造成的浪费。

### 2.2 产品定位

面向篮球爱好者的**按天实战篮球鞋租赁平台**，提供多品牌、多尺码、多打法的鞋款选择，支持在线选鞋、下单、支付、归还的全流程。

### 2.3 产品目标

- 降低用户尝试高端篮球鞋的成本；
- 为平台方提供鞋款库存管理和订单运营工具；
- 通过 AR 试穿等差异化功能提升下单转化率。

---

## 3. 用户角色

| 角色 | 描述 | 核心诉求 |
|---|---|---|
| **普通用户** | 需要租赁篮球鞋的篮球爱好者 | 快速找到合适鞋款、清晰的价格和库存、便捷的下单流程 |
| **平台管理员** | 运营人员/店主 | 管理鞋款和库存、跟踪订单状态、查看经营数据 |
| **系统** | 后端服务 | 保证数据一致性、库存扣减正确、订单状态流转正常 |

## 4. 功能需求

### 4.1 用户端

#### 4.1.1 首页与鞋库

- 展示平台所有上架鞋款；
- 支持按关键词搜索（鞋名、品牌、标签）；
- 支持按分类筛选（如后卫速度型、锋线支撑型、全能缓震型、实战耐磨型）；
- 显示每款鞋的库存总数、可租尺码、日租金、押金、评分。

#### 4.1.2 鞋款详情与下单

- 展示鞋款名称、品牌、分类、描述、标签、评分、图片；
- 列出各尺码库存状态；
- 用户选择尺码、租赁开始日期、结束日期；
- 自动计算租期天数、租金小计、押金、订单总额；
- 填写收货人、联系电话、配送地址；
- 提交订单并模拟支付。

**业务规则**：

- 开始日期不能早于今天；
- 结束日期不能早于开始日期；
- 单次租赁最长 30 天；
- 下单时校验所选尺码可用库存；
- 下单成功后扣减对应尺码可用库存。

#### 4.1.3 我的订单

- 展示当前用户的所有订单；
- 显示订单号、鞋款信息、尺码、租期、订单金额、当前状态；
- 状态标签：已支付待发货、已发货、租赁中、已归还、已完成、已取消；
- 对于“已支付待发货”和“已发货”状态的订单，支持取消订单。

#### 4.1.4 AR 试穿

- 用户可上传脚部照片或开启摄像头；
- 将选中的鞋款图像叠加到脚部照片上；
- 支持手动拖动、缩放、旋转鞋款位置；
- 保存试穿效果图。

#### 4.1.5 用户认证

- 用户注册：姓名、邮箱、密码（至少 6 位）；
- 用户登录：邮箱 + 密码；
- JWT Token 认证，Token 有效期 7 天；
- 登录状态持久化（localStorage）。

---

### 4.2 管理端

#### 4.2.1 运营概览

- 上架鞋款总数；
- 进行中订单数（已支付、已发货、租赁中）；
- 累计租金收入（不含已取消订单）；
- 低库存尺码预警（可用库存 ≤ 1）。

#### 4.2.2 订单管理

- 查看平台全部订单；
- 显示订单号、用户信息、鞋款、尺码、租期、金额、状态；
- 管理员可修改订单状态，支持状态流转；
- 订单进入终态（已归还 / 已完成 / 已取消）后自动释放库存。

#### 4.2.3 鞋款管理

- 新增鞋款：名称、品牌、分类、描述、图片、日租金、押金、评分、标签、尺码库存；
- 编辑鞋款：修改上述字段、上架/下架；
- 维护各尺码总库存和可用库存；
- 鞋款下架后不在用户端展示。

## 5. 订单状态机

```
                    ┌─────────────┐
         取消       │   已支付    │────┐
    ┌──────────────▶│  待发货     │    │ 发货
    │               └──────┬──────┘    ▼
    │                      │      ┌─────────┐
    │                      │      │ 已发货  │
    │                      │      └────┬────┘
    │                      │           │ 用户签收
    │                      │           ▼
    │                      │      ┌─────────┐
    │                      │      │ 租赁中  │
    │                      │      └────┬────┘
    │                      │           │ 归还
    │                      │           ▼
    │                      │      ┌─────────┐
    │                      │      │ 已归还  │
    │                      │      └────┬────┘
    │                      │           │ 验收完成
    │                      │           ▼
    │                      │      ┌─────────┐
    └──────────────────────┘      │ 已完成  │
                                  └─────────┘
```

**终态**：已归还、已完成、已取消。进入终态后释放库存。

## 6. 非功能性需求

### 6.1 性能

- 首页加载时间 < 2 秒（在常规网络下）；
- API 响应时间 < 500ms（不含网络延迟）；
- 静态资源支持浏览器缓存。

### 6.2 安全

- 用户密码使用 bcrypt 哈希存储；
- JWT Token 用于接口鉴权；
- 管理员接口需要 admin 角色权限；
- 数据库文件不暴露到公网；
- 生产环境使用 HTTPS。

### 6.3 兼容性

- 支持 Chrome、Firefox、Safari、Edge 等主流浏览器；
- 支持桌面端和移动端基础访问；
- AR 试穿功能依赖摄像头权限，需兼容 HTTPS 环境。

### 6.4 可用性

- 后端服务使用 systemd 常驻，支持开机自启；
- Nginx 作为反向代理和静态资源服务器；
- SSL 证书使用 Let's Encrypt，支持自动续期。
